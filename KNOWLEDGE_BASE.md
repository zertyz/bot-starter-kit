# Knowledge Base

## First-level directories and files Index
1. File `KNOWLEDGE_BASE.md` (this file) contains an Index for AI to quickly navigate this project.
2. File `AGENTS.md` contains Codex-specific instructions.
3. File `CLAUDE.md` contains Claude-specific instructions.
4. File `CODE_GUIDELINES.md` contains code compliance definitions.
5. File `CODE_PATTERNS.md` contains project-specific implementation patterns agents should honor before changing code.
6. File `Cargo.toml` contains this Rust project's definitions.
7. Directory 'management' contains requirements, progress, and rules about how this project is managed and should be conducted.
8. Directory `.github` contains the GitHub workflows.
9. Directory `scripts` contains helper scripts for various parts of this project.
10. Directory `diagrams` contains images used in the Rust docs, to visually aid the learning-curve for this project.
11. Inside `management`, file `BUGS.md` records active unresolved intake under `Open Reports` and may retain inactive addressed reports in other level-two sections.
12. Inside `management`, file `DECISIONS.md` records accepted project decisions.
13. Inside `management`, file `DEFINITION_OF_READY_DONE.md` records state-transition readiness and completion rules.
14. Inside `management`, file `RELEASES.md` records production release decisions.
15. Inside `management`, file `RISKS.md` records active and closed project risks.
16. Inside `management`, file `INCIDENTS.md` records operational, security, data, release, and staging incidents.
17. Inside `management`, file `EXPERIMENTS.md` records spikes, prototypes, and experimental branches.
18. Inside `management`, file `TRACEABILITY.md` records requirement-to-evidence links.
19. Inside `management`, file `TOOLING.md` explains the repo-local management helper commands.
20. Inside `management`, file `README.ai.md` is the fast-path operating map for AI agents revisiting the management governance system.
21. Directory `.agents/skills/analyze-management` contains the repo-local semantic management-analysis workflow; deterministic context comes from `scripts/management/semantic_context`.
